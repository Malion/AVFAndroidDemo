Week 1

These are ideas and concepts for my video game review app. I went through the list of supported API's and decided on the ones that I felt would best fit my app and the intent of my app. As each is implimented there will either be an example posted inside of the collapsible object or a link to show where it is.

Storage

I would like to use this API in order to store information rather than pull it fresh from the server every startup. Incase there is an issue with internet connection the app will still function and give information until it is next connected and updated.

Geolocation

Having this be a two part API in a sense where the phone will pull geolocation and date / time it will allow me to impliment date, time and location of the newly added entry's into my app

In App Browser

This functionality would be very useful because if there is a game that you would like to purchase because you enjoyed the review on it I could add some icons on the bottom of each review and have the site send the user to there chosen site. i.e. gamestop or amazon. This could all be done in app.

Notification

This is highly important, so as a user you want to be up on the latest and greatest. This function will allow you to keep up to date minute by minute if a new review is posted you will know about it because your device will send you a push notification.

Contacts

This API would be great because there is always some sort of share option. You could share it through your phones contacts and it could help spread the app.

Research

Week 2 Discussion Post

Using twitter for login and payment methods would make a person feel more at ease especially if in order to login to twitter it switched to the twitter app for verification and if only the required code from twitter was given to said app. This is used in a similar manor in a lot of apps now. Many of the games that are free to play on Android and iOS allow for market transactions. The way this is done is when you are ready to purchase something in the market the app goes into the background and brings up the market app for purchasing the product once completed the store sends verification back to the app that the transaction is completed successfully and you receive your product. So long as the end result is that the app is not forcing you to put your information into itself, many people will trust this system a lot more because it is a new application.

JSON-P is used for cross-domain reference. Say for instance you want to access data on a site if the site is JSON-P enabled then it will have a reference to this capability you would then input the reference into the address bar and give it a name. The site will then understand your request and send back a JSON object which you can use a simple code to restructure into whatever you need it for. This could be used to make a list of items of all kinds from a library resource to connecting to a bank site to pull recent transactions.

Flicker API could be used with the photo album to upload and download pictures into folders on the device or you could use the service as a photo cloud so that your pictures were always backed up.

I am looking forward to being able to hook up to the web to pull information from domains and utilize it in an application as well as being able to use the devices native features. For this I want to see a few things come from my app. Such as the ability to cross communicate from a domain into the app and then separate data accordingly into a calendar, media library, and notifications. Because pop-up notifications can be really irritating this will be secondary or just not used at all instead I will utilize messages displayed and created on the page dynamically.

Personally I have learned a lot about doing research on my assignments because I do enjoy programming I try to make my app’s do more than they are asked to do and with the aid of research can generally accomplish this goal. However it can be extremely costly when it comes to time. A few projects have taken me 60+ hours in a week to complete but the end result has always paid off in my opinion.
